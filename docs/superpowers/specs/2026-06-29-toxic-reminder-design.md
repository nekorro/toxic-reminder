# toxic-reminder — дизайн

Дата: 2026-06-29

## Проблема

Уведомление о встрече приходит за 5 минут до начала. За эти 5 минут легко
отвлечься на другую задачу и забыть про встречу — в итоге встречи пропускаются.

## Цель

В **момент начала встречи** показать заметную (яркую красную) плашку поверх всех
окон с двумя кнопками:

- **«Скрыть»** — убирает плашку.
- **«Подключиться»** — открывает ссылку на видеовстречу из события
  (например `https://telemost.yandex.ru/j/0429743918`).

Плашка показывается **только для событий, у которых есть видеоссылка**.

## Принятые решения

| Решение | Выбор |
|---------|-------|
| Источник данных | Напрямую по CalDAV (`caldav.yandex-team.ru`) |
| Формат приложения | Меню-бар приложение + плашка поверх всего |
| Стек | Python: `rumps` (меню-бар) + `PyObjC` (плашка) + `caldav`/`icalendar` |
| Хранение секретов | macOS Keychain (пароль), логин — в конфиге |
| Когда показывать плашку | Только события с видеоссылкой |
| Меню-бар | Минимум + строка ближайшей встречи (дата, время, название) |

## Архитектура и компоненты

Система разбита на изолированные модули с одной ответственностью каждый.
Вся логика вынесена из UI, чтобы её можно было тестировать независимо.

### `config.py`
Чтение/запись `~/.toxic-reminder/config.json`:
- `login` — `ваш-логин@yandex-team.ru`
- `server_url` — `https://caldav.yandex-team.ru`
- `refresh_interval_sec` — интервал фонового обновления (по умолчанию 300)
- `lookahead_hours` — окно выборки событий вперёд (по умолчанию 24)

Зависимости: только stdlib (`json`, `pathlib`).

### `keychain.py`
Доступ к доменному паролю в macOS Keychain через CLI `security`:
- `get_password(account) -> str | None`
- `set_password(account, password)`

Логин хранится в конфиге, пароль — только в связке ключей (в открытом виде не лежит).
Зависимости: `subprocess`.

### `caldav_client.py`
- Подключается к `server_url` с `login` + паролем из Keychain.
- Выгружает события на окно `lookahead_hours` вперёд.
- Возвращает `list[Event]`, где `Event = (uid, title, start: datetime, link: str | None)`.
- Ссылку извлекает через `link_extractor`.

Зависимости: `caldav`, `icalendar`, `keychain`, `link_extractor`.

### `link_extractor.py`
**Чистая функция без сети** — сердце фильтра «только с видеоссылкой».
Вход: поля VEVENT (`URL`, `LOCATION`, `DESCRIPTION`, `CONFERENCE`, `X-*`).
Выход: первая найденная ссылка на видеовстречу или `None`.

Распознаёт по regex известных провайдеров (Telemost `telemost.yandex.ru`,
Google Meet, Zoom, Teams и общий `https://`-URL в полях `URL`/`CONFERENCE`).
Покрывается тестами плотнее всего.

Зависимости: `re` (stdlib).

### `scheduler.py`
- Держит таймеры на момент `start` каждого события, у которого есть ссылка.
- Дедуп по `uid` — плашка для одного события показывается один раз.
- Периодически (по `refresh_interval_sec`) просит `caldav_client` обновить список
  и переустанавливает таймеры.
- При срабатывании таймера вызывает колбэк показа плашки.

Зависимости: `caldav_client`, стандартные таймеры (`threading.Timer` или таймер rumps).

### `banner.py`
PyObjC borderless `NSWindow`:
- Высокий window level (поверх всего), показ на всех Spaces.
- Красный фон, крупный текст: название встречи + время.
- Кнопка **«Подключиться»** → открывает ссылку через `NSWorkspace`, закрывает окно.
- Кнопка **«Скрыть»** → закрывает окно.

Зависимости: `PyObjC` (`AppKit`).

### `app.py`
`rumps`-приложение меню-бара:
- Иконка в меню-баре.
- Первая строка меню — ближайшая встреча: `ДД.ММ ЧЧ:ММ — Название`.
- Пункты: «Обновить», «Задать пароль», «Выход».
- Состояние ошибки (нет пароля / неверные креды) отражается в иконке/строке.
- Владеет `scheduler`, по колбэку таймера показывает `banner`.

Зависимости: `rumps`, `scheduler`, `banner`, `config`, `keychain`.

### LaunchAgent plist
`~/Library/LaunchAgents/ru.nekorro.toxic-reminder.plist` — автозапуск через `launchd`
при логине пользователя.

## Поток данных

```
launchd → app стартует
       → config + пароль из Keychain
       → caldav_client тянет события (lookahead_hours)
       → link_extractor оставляет только события со ссылкой
       → scheduler ставит таймер на start каждого (дедуп по uid)
       → таймер сработал → banner показан
            ├─ «Подключиться» → открыть ссылку → закрыть
            └─ «Скрыть» → закрыть
       → фоновый рефреш каждые refresh_interval_sec → перечитать → переустановить таймеры
```

## Обработка ошибок

- **Нет сети** → используем последний известный список событий, тихий ретрай на
  следующем рефреше.
- **Нет пароля в Keychain / неверные креды** → строка меню в состоянии ошибки +
  пункт «Задать пароль» для ввода.
- **Событие без ссылки** → плашку не показываем вовсе.
- **Дубль события** (повтор в рефреше) → не показываем повторно (дедуп по `uid`).

## Тестирование

- `link_extractor` — много кейсов: Telemost в `URL`, в `DESCRIPTION`, в `LOCATION`,
  несколько ссылок, отсутствие ссылки, мусорные URL.
- `caldav_client` — парсинг на фикстурах `.ics`.
- `scheduler` — дедуп по `uid` и постановка таймеров с мок-временем.
- `keychain` — с моком `subprocess`.
- `banner` / `app` (UI) — проверяются вручную, юнит-тестами не покрываются.

## Вне рамок (YAGNI)

- Чтение через системный Календарь/EventKit (выбран прямой CalDAV).
- Плашка для встреч без видеоссылки.
- Полноценное окно настроек (хватает конфиг-файла + пункта «Задать пароль»).
- Поддержка нескольких аккаунтов.
